If you are into front-end development, and you started reading about TypeScript vs JavaScript, minification, css preprocessor, task runner and bundling, but you don't want to mess with a big framework, you came to the right place.

Here is presented a website, used as a base for your upcoming projects that need to be pure or frameworkless with the scalable advantages that TS and Sass offer, so you can spend less time configuring and more time coding.

Why Sass?

SASS is a CSS preprocessor, it lets you generate Css from Sass unique syntax allowing to make more with less code, using variables, nesting, partials, mixins and much more!

What about TypeScript?

TypeScript extends JavaScript by adding types. By understanding JavaScript, TypeScript saves you time catching errors and providing fixes before you run code. Also, Typescript mixes the old with the new in the best way possibility. You can use newer features from ES6, and beyond, and the compiler will convert them to ES5 (or whatever you specify). This allows you to make use of newer features that will be the future of Javascript, without having to worry about compatibility. The finished code produced by the compiler will be backwards compatible with ES5 and will work just fine with older browsers.

Ok, so what is Gulp?

Gulp is a task runner that provides automation for repetitive workflows and compose them into efficient build pipelines, so we can have a confortable developing environment and automated tasks for deployment in different environments.
